Rules to Follow While Writing HTML

General Rules

  • Start the document with <!DOCTYPE html> to define the document type as HTML5.
  • Use proper HTML structure: <html>, <head>, and <body> sections.
  • Always declare the lang attribute in the <html> tag for accessibility.

Syntax Rules

  • Use lowercase for HTML tags and attributes.
  • Always close tags (including self-closing tags like <img /> and <input />).
  • Enclose attribute values in double quotes (").
  • Avoid using deprecated tags (e.g., <font>, <center>, <marquee>).

Indentation and Nesting

  • Use consistent indentation (2 or 4 spaces per level).
  • Nest elements properly to maintain a clear hierarchy.
  • Do not overlap or improperly nest tags.

Semantic HTML

  • Use semantic tags like <header>, <footer>, <nav>, <article>, and <section>.
  • Use heading tags (<h1> to <h6>) in a logical and hierarchical order.

Accessibility

  • Provide descriptive alt attributes for images.
  • Use <label> elements for form controls.
  • Use landmarks like <main>, <nav>, and <footer>.
  • Ensure proper contrast between text and background colors.

Attributes

  • Use unique id attributes and meaningful class names.
  • Use appropriate attributes like href for links and src for images.
  • Avoid inline styles and scripts whenever possible.

Comments

  • Use comments (<!-- Comment -->) to explain sections of the code, but avoid excessive commenting.

External Resources

  • Link external CSS files using <link> in the <head> section.
  • Place external JavaScript files at the bottom of the <body> or use defer/async.

Performance and Optimization

  • Minimize inline CSS and JavaScript.
  • Compress and optimize images for faster loading.
  • Use meta tags for viewport settings to make the website responsive.

Validation

  • Validate your code using the W3C Validator to ensure standards compliance.
  • Avoid duplicate id attributes.

Avoid Common Mistakes

  • Do not use inline CSS styles (<p style="color: red;">).
  • Avoid using too many <div> and <span> tags unnecessarily.
  • Ensure all <a> tags have meaningful text, not just “Click here.”
